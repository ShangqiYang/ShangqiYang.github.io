Dynamic Programming Summary

发表于 2018-08-01

DP简单认识

  • 动态规划是一种算法思想，可以理解为根据之前的信息，推导出现在的状态！
  • 推导过程中，记录每次得到的状态，为之后状态的推导提供信息。
  • 因为记录了每次得到的信息，所以减少了重复状态的计算，速度加快。
  • 动态规划有方向性，不回头，不绕圈，无循环依赖。

DP组成部分(SFIO四要素)

  1. 状态(State):存储小规模问题的结果
    • 最后一步
    • 转化为子问题
  2. 方程(Function):如何从前一种状态到现在状态
    • 寻找每个状态之间的联系
  3. 初始化(Init & Corner)
    • 最极限的小状态，起点
    • 边界情况
  4. 计算顺序(Order):
    • 计算每种状态的顺序，有依赖关系
    • 如何利用之前的计算结果

DP适用情况

  • 适合把暴力时间复杂度(指数型)转化为多项式型的。(O(2^N),O(N!)转化为O(N^2),O(N^3))
    1. 统计方案个数
      • 有多少种方式可以走到右下角
      • 有多少种方法可以选出K个数字使和为Sum
    1. 最大最小值
      • 最长上升子序列的长度
      • 从左上角走到右下角路径的最大数字和
    1. 判断是否可行(存在性)
      • 能不能选出K个数字使和为Sum
      • 博弈游戏，先手是否可以获胜

DP不适用情况

  • 问题的时间复杂度已经在多项式级别(O(N^2),O(N^3)）
  • 问题要求的是所有的，具体的方案(而不是方案个数)，需要用Recursion
  • 问题输入的数据是一个集合，而非序列()

DP分类

  • 坐标型DP(20%)
  • 序列型DP(20%)
  • 划分型DP(20%)
  • 区间型DP(15%)
  • 背包型DP(10%)
  • 最长序列型DP(5%)
  • 博弈型DP(5%)
  • 综合型DP(5%)

DP Follow Up

  • 空间优化
  • 打印路径

DP例题

1. Coin Change

You are given coins of different denominations and a total amount of money amount. Write a function to compute the fewest number of coins that you need to make up that amount. If that amount of money cannot be made up by any combination of the coins, return -1.

Example 1:
coins = [1, 2, 5], amount = 11
return 3 (11 = 5 + 5 + 1)

Example 2:
coins = [2], amount = 3
return -1.
